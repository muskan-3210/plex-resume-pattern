plex-resume-pattern

I built an ATS-friendly LaTeX resume template for software engineers.

After trying multiple templates, I realized most of them:

* Break ATS parsing
* Waste space
* Focus too much on design, not impact

So I created a clean, minimal template focused on:
• Impact-driven bullet points
• Strong engineering keywords
• Recruiter-friendly structure

If you're applying to software roles (especially FAANG, MAANG-level), this might help.
